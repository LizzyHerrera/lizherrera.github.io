01 — About

Who I Am

I'm a Character Technical Artist specializing in rigging systems, OpenUSD asset pipelines, and production tooling for large-scale avatar ecosystems. Based in Los Angeles.

At Genies, I defined the Gen13 Avatar Standard — now the canonical structure for all production avatars — and built the systems that power how characters are rigged, exported, and shipped across the platform.

I work at the intersection of art, engineering, and ML: designing pipeline architecture, building Maya and Houdini tools that artists actually want to use, and delivering datasets that feed machine learning models.

I care about clean deformation, modular systems, and tools that don't get in the way of creativity.

01 — Avatar Standard
Gen13 Avatar Standard

Defined the canonical avatar structure used across all Genies production — skeleton, OpenXR facial blendshape schema, and layered USD asset hierarchy. The spec all downstream tooling, CI, and rendering depends on.

  • Established USD assetInfo metadata schema as pipeline-wide canonical identifier
  • Migrated ARKit 52-blendshape set → full OpenXR Face Tracking 2 expression set
  • Led batch migration of 1,800+ legacy wearable assets to the new standard
  • Authored 150 USD template files for humanoid and mammal asset types
Stack: OpenUSD (USDA), OpenXR Face Tracking 2, Maya, Python

02 — Pipeline Tool
Rapid Companion Tool + USD Export API

End-to-end Maya rigging companion covering the full avatar workflow — mesh loading through USD export and CI/CD dispatch — without leaving Maya. Built alongside a modular USD export API that powers the export layer.

  • PySide2/6 hub: mesh load → skeleton bind → skinning → blendshapes → export → publish
  • Modular export API producing layered avatar.usd/usdz with independently-skippable steps
  • Sparse iteration layer system for non-destructive artist overrides on a published base
  • GitHub Actions dispatch embedded — trigger and monitor CI jobs without leaving Maya
Stack: Python, PySide2/6, OpenUSD, MayaUSD, GitHub Actions API

03 — ML Data Systems
Facial Landmarks + Synthetic Geometry

Designed the 3D facial landmark specification for humanoid and mammal characters stored directly in USD, and built a Houdini TOPS pipeline generating thousands of synthetic head variants for ML training.

  • Landmark spec encoded as UsdGeom.Points: jaw, brow, eye contour, lip, pupil for human and mammal
  • Extended COCO to 57 joints with full bilateral hand skeleton — delivered as ML training ground truth
  • Houdini TOPS: 6,000+ facial variants from 500 head targets across 5 ethnic groups, fully batched
Stack: Houdini TOPS, OpenUSD, Python (hou API), OpenCV, NumPy, COCO keypoints

04 — Tooling
Production Maya Toolset

Suite of Maya/PySide tools built to solve specific friction points for the rigging and technical art team — each shipped and used in production.

  • UAPT: Batch wearable refit pipeline with dry-run validation, JSON asset metadata autofill, and QC turntable
  • Material → USD Converter: Any shader network to usdPreviewSurface with on-the-fly ORM texture baking
  • Blendshape Calibrator: Normalize blend targets — dual list/tree view, Maya 2024/2025 compatible
Stack: Python, maya.cmds, MayaUSD, PySide2/6, OpenCV, NumPy
