About

I'm a Character Technical Artist specializing in rigging systems, OpenUSD asset pipelines, and production tooling for large-scale avatar ecosystems. Based in Los Angeles.

At Genies, I defined the Gen13 Avatar Standard — now the canonical structure for all production avatars — and built the systems that power how characters are rigged, exported, and shipped across the platform.

I work at the intersection of art, engineering, and ML: designing pipeline architecture, building Maya/Houdini tools that artists actually want to use, and delivering datasets that feed machine learning models.

I care about clean deformation, modular systems, and tooling that scales.

PROJECT 01 — AVATAR STANDARD
Gen13 Avatar Standard

Pipeline diagram — add asset here

Defined the canonical avatar standard used across all Genies production. Established skeleton, USD layer hierarchy, and OpenXR facial blendshape schema — the spec all downstream tooling depends on.

  • Defined USD assetInfo metadata schema used as canonical identifier pipeline-wide
  • Migrated ARKit 52-blendshape set → full OpenXR Face Tracking 2 expression set

  • Authored 150 USD template files for humanoid and mammal asset types

PROJECT 02 — PIPELINE TOOL
Rapid Companion Tool + USD Export API
⬡
Tool demo — add screen recording here

End-to-end Maya rigging companion covering the full avatar workflow — mesh loading through USD export and CI/CD pipeline dispatch — without leaving Maya.

  • PySide2/6 tool: mesh load → skeleton bind → skinning → blendshapes → export → publish
  • Modular USD export API producing layered avatar.usd/usdz packages
  • Sparse iteration layer system for non-destructive artist overrides on a published base
  • GitHub Actions dispatch embedded so artists can trigger and monitor CI jobs in-tool
Stack: Python, PySide2/6, OpenUSD, MayaUSD, GitHub Actions API
PROJECT 03 — ML DATA SYSTEMS
Facial Landmarks + Synthetic Geometry
⬡
Landmark diagram — add asset here

Designed the 3D facial landmark specification for humanoid and mammal characters stored directly in USD, and built a Houdini TOPS pipeline to generate thousands of synthetic head geometry variants for ML training.

  • Landmark spec: jaw contour, brow, eye contour, lip, pupil encoded as UsdGeom.Points in USD
  • Extended COCO to 57 joints with full bilateral hand skeleton
  • Houdini TOPS network: 6,000+ facial variants from 500 heads × 5 ethnic groups, full batch
  • Delivered ground truth data for Genies humanoid and mammal face detection models
Stack: Houdini TOPS, OpenUSD, Python (hou API), OpenCV, NumPy, COCO keypoints
PROJECT 04 — SUPPORTING TOOLS
Production Maya Toolset
